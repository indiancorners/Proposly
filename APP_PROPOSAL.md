## 06. Terms & Conditions

### Scope of Work
- This quote covers design, UX/UI, prototype, and documentation only.
- Scope includes [N] screens delivered in phases.
- **Changes in screen count or feature scope after Phase 1 sign-off are treated as change requests and billed at [RATE] per screen or [RATE] per feature flow.**

### Exclusions
- Development (frontend, backend, APIs, database, hosting) is explicitly excluded.
- Handoff limited to Figma file, exports, and design documentation.
- Implementation support beyond basic file explanation is not included.

### Revisions
- Maximum **2 rounds per phase** unless otherwise agreed.
- Design refinements, color/type, layout adjustments = in scope.
- Feature additions, major flow changes, new screens = change requests.

### Ownership & Handover
- All deliverables transfer to client upon receipt of final payment.
- Figma file handed over organized, documented, and dev-ready.

### Change Request Protocol
1. Document the change (screen, flow, direction shift)
2. Estimate impact (hours, timeline extension)
3. Provide change request cost in writing
4. Client approves before work begins
5. New cost + timeline documented

### Validity
- Quote valid for **14 days** from date of issue.
- Project commences upon written confirmation + Tranche 1 payment.
